# GLASS Examples

This directory contains example outputs from the **GLASS (Global Long-context Agent Scoring System)** evaluation harness and research pipeline. These artifacts demonstrate the transparency and depth of analysis provided by the system.

## Directory Structure

- **[runs/](./runs/)**: Contains raw and processed artifacts from individual evaluation runs. This includes:
  - `manifest.json`: Reproducibility metadata (git hashes, library versions).
  - `config.yaml`: The exact configuration used for the run.
  - `results.csv`: Per-sample performance data.
  - `summary.md`: Aggregate performance highlights.
  - `statistics.json`: Rigorous statistical analysis (Bootstrap CIs, p-values).
- **[research_insights/](./research_insights/)**: Contains synthesized research reports generated by the Research Harness. This includes:
  - `figures/`: Distribution-aware visualizations (Violin, Forest CIs, Domain Heatmaps).
  - `insights.md`: LLM-generated qualitative synthesis and error analysis grounded in paired divergences.
  - `aggregated_data.json`: Cross-run data extracted for scientific comparison.

> [!NOTE]
> The `logs/` directory, which typically contains the raw request text sent to the LLM during synthesis and vision interpretation, has been omitted from these examples to keep the artifacts concise.

## Purpose

These examples serve several purposes:
1. **Auditability**: Demonstrate the "transparent-box" design where every decision and artifact is preserved.
2. **Reference**: Provide a template for researchers to understand the expected output format.
3. **Verification**: Allow users to verify the harness's behavior with pre-computed results.

For instructions on how to generate your own runs and insights, see the core **[GUIDE.md](../GUIDE.md)**.
